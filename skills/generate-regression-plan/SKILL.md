# Generate Regression Plan — 生成回归计划

## Why — 为什么需要
一次改动后，不可能把所有用例重跑一遍（时间、成本）。贵的错误有两个方向：**过度回归**（全量重跑，成本失控）或**砍错了子集**（省下时间却漏了真正会回归的那块）。

回归规划的本质是**基于风险的选择**：给定一次改动，决定**测什么、跳什么**，并且**为"跳"留痕**。它把"凭感觉估影响面"换成可追溯的影响分析——哪些区域必测、哪些可跳、跳的依据是什么、残余风险是什么。

## When — 何时使用
**用它当：**
- 有一次明确改动（PR / 功能 / 修复）+ 一份可选择的面（功能图 / 用例清单 / 依赖关系），要在预算内排出回归范围
- 需要给出"这个计划没覆盖什么、风险多大"的可交代结论

**不要用它当：**
- 改动没界定清楚（回归相对于什么？）→ 先拿到 diff / 改动范围
- 全新功能、无既有面可选 → 那是新用例设计，用 `generate-test-case`
- 反正要全量跑一遍 → 不存在"选择"问题，本 Skill 无用武之地

## Input — 输入契约
**必需输入：**
- **改动**：改了哪些代码 / 配置 / 数据，及其意图
- **可选择的面**：功能图 / 既有用例清单 / 依赖关系
- **约束**：时间 / 资源预算（或"只排优先级、无硬上限"）

**前置条件：** 改动可界定（说得清改了什么）+ 存在可选择的面。

**异常输入的处理：**

| 输入缺陷 | 处理 |
|---|---|
| 改动没界定（不知道改了啥） | 停。要 diff / 改动清单，否则无法做影响分析 |
| 无功能图 / 用例清单 | 退回索要；或降级到功能级粗粒度图，并标注粒度受限 |
| 给了预算但 must-run 已超预算 | **不静默压缩**——照排，把"超预算"作为风险信号上报 |

## Process — 执行流程
一条风险优先的选择流水线。

1. **影响分析（blast radius）**：从改动出发，追**直接改到的模块 + 下游依赖 + 共享数据/状态**。不停在"改了哪个文件"。
2. **风险打分**：每个候选区域按 **可能性（是否被改到/强耦合）× 影响（是否用户可见/核心路径/数据完整性）** 打分。
3. **分层**：`必测（must）/ 应测（should）/ 跳过（skip）`。
4. **预算装箱**：在预算内从 must→should 填。**must-run 超预算 → 上报，不是偷偷删到够。**
5. **为"跳"留痕**：skip 里每一条都写明**为什么跳**（不在影响范围 / 风险低 / 已被别处覆盖）。这是本 Skill 的核心纪律——见 [SPECIFICATION 的"反静默省略"](../../SPECIFICATION.md)。
6. **锁定专项回归**：本次若是修 bug，把该 bug 的复现路径转成**永久回归用例**。
7. **成表输出**：分层计划 + 显式 skip 清单 + 残余风险声明。

> 关键判断点：
> - blast radius 按**耦合**判，不按**改动行数**判——小改动动到共享/核心路径，影响面照样大。
> - "预算不够"不是砍测试的理由，是**上报风险**的信号。

## Output — 完成定义

**输出契约（固定 Schema）：** 分层计划表 + skip 清单 + 残余风险。

| 区域 | 层级 | 风险（可能性 × 影响） | 依据（为什么这个层级） | 关联改动 |
|---|---|---|---|---|

- `层级` 枚举：`必测 | 应测 | 跳过`
- **skip 清单**：所有"跳过"的区域单列，每条带跳过理由。
- **残余风险声明**：这个计划**没覆盖什么**、意味着什么（一句可交代的话）。

**决定性规则（保证可复现）：**
- **风险打分**：可能性/影响各分 高/中/低，组合定层级（高×高=必测，低×低=跳过）。
- **排序**：先按层级 `必测→应测→跳过`，同层级内按风险从高到低。
- **禁止静默截断**：因预算未覆盖的区域，必须出现在 skip 清单并注明"预算不足"，不得消失。

**完成标准（逐条可核）：**
- [ ] blast radius 含下游依赖与共享状态，不止改动点本身
- [ ] 每个区域有风险分与层级依据
- [ ] skip 清单每条带理由；因预算跳过的显式标注
- [ ] 有残余风险声明
- [ ] 若为修 bug，其复现已转为永久回归用例

## Failure Patterns — 失败模式
| 现象 | 常见误判 | 正确判断路径 |
|---|---|---|
| 只测改动的文件本身 | 以为够了 | 追下游依赖与共享状态；blast radius 不止改动点 |
| 时间不够就砍低优先级 | 静默砍掉，漏测无据可查 | skip 必须留痕带理由（反静默省略） |
| must-run 超预算 | 硬删到塞进预算 | 这是风险信号，应上报，而非偷偷压缩 |
| 改动看着很小 | 判影响面小 | 按耦合判不按行数判；小改动动核心/共享照样大范围 |
| 修完 bug 没加进回归 | 当次修好就完事 | 每个修复的复现转成永久回归用例，防复发 |
| 全量回归最安全 | 无脑全测 | 成本失控；用风险优先子集 + 残余风险声明替代 |
