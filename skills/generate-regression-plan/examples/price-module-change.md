# Example: 改动共享"价格计算"模块

一个刻意通用、不含任何具体项目信息的样例，展示 `generate-regression-plan` 怎么按风险选子集并为跳过留痕。（与 `analyze-bug` 的价格 null bug 同源，演示修复后的回归规划。）

## 输入
- **改动**：修复"缓存回退返回 null 价格"，在价格计算模块的回退路径加了 null 校验。
- **面**：下单 / 购物车 / 收据 / 促销 / 后台价格报表 —— 均消费价格模块。
- **约束**：回归预算约 4 小时。

## 输出（按输出契约）

### 分层计划

| 区域 | 层级 | 风险（可能性 × 影响） | 依据 | 关联改动 |
|---|---|---|---|---|
| 下单主流程 | 必测 | 高 × 高 | 直接消费价格、核心交易路径 | 回退路径改动 |
| 购物车金额 | 必测 | 高 × 高 | 实时算价、用户可见 | 同上 |
| 收据/发票金额 | 应测 | 中 × 高 | 依赖最终价，非实时但影响对账 | 同上 |
| 促销/折扣计算 | 应测 | 中 × 中 | 叠加价格，耦合中等 | 同上 |
| 专项：超时+缓存未命中 | 必测 | — | 原 bug 复现，转永久回归用例 | 本次修复 |

### skip 清单（每条带理由）
| 区域 | 跳过理由 |
|---|---|
| 后台价格报表 | 只读展示、不参与算价路径，风险低 |
| 历史订单价格快照 | 已落库的历史值，不重算，不在 blast radius |

### 残余风险声明
> 本计划未覆盖后台报表与历史快照的展示层。若价格模块的**输出格式**（非数值逻辑）有变，报表展示可能受影响——本次改动不涉及格式，故判残余风险低。

## 这个样例演示了什么
- **blast radius 按耦合追**：一个"加 null 校验"的小改动，波及 5 个消费方，不是只测改的那个文件。
- **skip 留痕**：后台报表、历史快照被跳过，但**写明了为什么**——不是消失，是显式决策（反静默省略）。
- **修复转永久回归**：原 bug 的复现路径进了必测,防同 bug 复发。
- **残余风险声明**：把"没覆盖的部分及其判断"摆到台面，让计划可交代。
